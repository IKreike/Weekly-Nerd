Declarative design

Jeremy (schrijver)

(datum)

There are two approaches to design in the workfield, declarative and imperative. Neither is wrong, neither is right, but we are gonna go over some highlights of both methods.

First, the basis [3]

With imperative design you work with precise instructions. With declarative design you work with the output you want and let the software think about how to get there.

How does this apply in the World Wide Web

  1. Html – declarative
  2. Css - declarative
  3. Js – imperative

This makes both html and css fault tolerant, while javascript stops execution if there is an error in the code

“javascript should only do what only javascript can do” [quote]

Why does it matter

This distinction between mindsets is, in the root, a difference in control. Why do you choose to make your own button, or to trust the browser to make one for you. Do you see the code the browser gives you as features or as bugs? My advise- let the browser work for you.
